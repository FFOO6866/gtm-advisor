# GTM Advisor

> AI-Powered Go-To-Market Advisory Platform for Singapore SMEs

**Version**: 0.1.0 | **Status**: MVP | **Python**: >=3.12

## Project Overview

GTM Advisor is a self-service agentic AI platform that provides go-to-market strategies and lead generation for Singapore SMEs. Built on Grace Framework patterns with 6 specialized AI agents that collaborate via A2A protocol.

## Architecture: Agent Team

```
User Input → GTM Strategist → [Market Intel, Competitor, Customer, Leads, Campaign] → Results
```

| Agent | Purpose | Data Sources |
|-------|---------|--------------|
| **GTM Strategist** | Orchestrates, gathers requirements | User context |
| **Market Intelligence** | Market trends, opportunities | Perplexity, NewsAPI, EODHD |
| **Competitor Analyst** | SWOT, positioning | Perplexity, EODHD |
| **Customer Profiler** | ICP, personas | Analysis synthesis |
| **Lead Hunter** | Real prospect identification | Perplexity, databases |
| **Campaign Architect** | Messaging, templates | All insights |

## Agent Development

All agents MUST:
1. Inherit from `BaseGTMAgent`
2. Implement PDCA methods: `_plan()`, `_do()`, `_check()`, `_act()`
3. Use real data sources (not just LLM generation)
4. Include confidence scoring
5. Load their domain knowledge pack in `_plan()` and inject it into `_do()`

```python
from agents.core.src.base_agent import BaseGTMAgent
from packages.knowledge.src.knowledge_mcp import get_knowledge_mcp

class MyAgent(BaseGTMAgent[ResultType]):
    async def _plan(self, task: str, context: dict) -> dict:
        """Plan execution strategy — load knowledge pack here."""
        context = context or {}
        # Load synthesized domain guides for this agent
        kmcp = get_knowledge_mcp()
        self._knowledge_pack = await kmcp.get_agent_knowledge_pack(
            agent_name="my-agent",   # kebab-case, must match _AGENT_GUIDE_MAP key
            task_context=task,
        )
        return {...}

    async def _do(self, plan: dict, context: dict) -> ResultType:
        """Execute plan — inject knowledge pack into LLM user message."""
        _knowledge_ctx = getattr(self, "_knowledge_pack", {}).get("formatted_injection", "")
        _knowledge_header = f"{_knowledge_ctx}\n\n---\n\n" if _knowledge_ctx else ""
        messages = [
            {"role": "system", "content": self.get_system_prompt()},
            {"role": "user", "content": f"{_knowledge_header}Your task prompt here..."},
        ]
        return await self._complete_structured(response_model=ResultType, messages=messages)

    async def _check(self, result: ResultType) -> float:
        """Return confidence 0-1. Base score is 0.2 — must earn via data quality."""
        pass

    async def _act(self, result: ResultType, confidence: float) -> ResultType:
        """Adjust if needed, return final."""
        pass
```

## Knowledge Base Setup

The agents' domain expertise comes from a 4-layer knowledge system:

| Layer | Source | Status |
|-------|--------|--------|
| **1 — Framework constants** | `packages/knowledge/src/frameworks.py` | Always available |
| **2 — Book index** | `packages/knowledge/src/book_index.py` | Always available |
| **3 — Qdrant book chunks** | 2,804 chunks from 14 marketing books | Requires `build_knowledge_base_books.py` |
| **4 — Synthesized domain guides** | `data/knowledge/guides/*.json` | Requires `synthesize_domain_guides.py` |

**Setup order (one-time):**
```bash
# Step 1: Extract and embed all books into Qdrant
uv run python scripts/build_knowledge_base_books.py

# Step 2: Synthesize 18 domain guides from the Qdrant chunks
uv run python scripts/synthesize_domain_guides.py

# Preview what will be synthesized without spending API credits
uv run python scripts/synthesize_domain_guides.py --dry-run

# Re-synthesize a single guide
uv run python scripts/synthesize_domain_guides.py --slug digital_awareness_campaign --force
```

Guides are stored in `data/knowledge/guides/` (git-ignored alongside the Qdrant chunks). Each developer must run the synthesis script locally after cloning. When no guides exist, agents fall back to Layer 1 framework constants — no errors, no code changes needed.

**Adding a new guide:**
1. Add an entry to `GUIDE_CATALOG` in `scripts/synthesize_domain_guides.py`
2. Add the slug to `_AGENT_GUIDE_MAP` in `packages/knowledge/src/knowledge_mcp.py`
3. Add relevance keywords to `_GUIDE_RELEVANCE_KEYWORDS` in the same file
4. Run `uv run python scripts/synthesize_domain_guides.py --slug your_new_slug`

## Dashboard UI

The dashboard uses React 18 + TypeScript, Framer Motion, Tailwind CSS with glassmorphism design, and an Aurora background gradient.

### Two-Tier Product Model

| Tier | Route | Auth | Layout |
|------|-------|------|--------|
| **Free / Teaser** | `/` | None required | Header + ConversationPanel + AgentNetwork + ResultsPanel. **No sidebar.** |
| **Paid** | `/today`, `/campaigns`, `/prospects`, `/content`, `/insights`, `/results`, `/approvals`, `/sequences`, `/workforce`, `/playbooks`, `/settings`, `/why-us` | JWT required | AppShell → Header + SidebarNav + Outlet |

Key rules:
- Teaser page **always** opens `OnboardingModal` on mount (`showOnboarding` initialises to `true`). No sessionStorage check.
- Teaser `<Header teaser />` prop forces `isAuthenticated=false` → upgrade button always shows, never sign-out.
- AppShell auth guard checks `localStorage.gtm_access_token`; redirects to `/login?next={path}` if absent.
- `ResultsPanel` CTAs are auth-aware: unauthenticated → "Unlock Full Dashboard"; authenticated → "Go to Dashboard" (→ `/today`).
- "Analysis" entry in SidebarNav points to `/` so authenticated users can start a new run; the always-true `showOnboarding` ensures the modal opens.

Key components:
- `AppShell.tsx` — auth guard, CompanyContext hydration, layout wrapper
- `SidebarNav.tsx` — paid-tier navigation (primary + operations + bottom)
- `AgentNetwork.tsx` — A2A agent activity visualization
- `ConversationPanel.tsx` — chat interface; "Start here" area is a button that re-opens OnboardingModal
- `ResultsPanel.tsx` — analysis results + auth-aware CTAs
- `OnboardingModal.tsx` — company input form

### Auth Endpoints (backend)
```
POST /api/v1/auth/login     → {access_token, refresh_token}
POST /api/v1/auth/register  → {access_token, refresh_token}
GET  /api/v1/companies/     → CompanyListResponse  (trailing slash required — no-slash redirects 307)
GET  /api/v1/companies/{id} → CompanyResponse
```

### AppShell Hydration
1. Reads `localStorage.gtm_company_id` **or** `sessionStorage.gtm_company_id` (teaser flow writes to sessionStorage)
2. If either present + token exists: sets `isHydrating=true` immediately (prevents flash of empty state)
3. Calls `GET /api/v1/companies/{id}` with Bearer token
4. On success: sets CompanyContext; on failure: clears `localStorage.gtm_company_id` only

### Known Backend Quirks
- `products` field: seed script stores `[{name, description, type}]` (dicts) but `CompanyResponse.products: list[str]`. Fixed in `company_to_response()` to coerce dict items to their `name` field.
- `fit_score` on leads is 0–100 integer; divide by 100 for display percentages.
- `GET /api/v1/companies/{id}/leads` returns `LeadListResponse` (`{leads, total, by_status, avg_fit_score}`), not a bare array.

## Pitfalls to Avoid

| Pitfall | Solution |
|---------|----------|
| Generic LLM responses | Use Perplexity/NewsAPI for real data |
| No confidence scoring | Always implement `_check()` properly |
| Missing PDPA compliance | Enable governance in production |
| Hardcoded company data | Use actual search/research |
| UI not showing agent activity | Check AgentNetwork connections |
| Domain guides not loading | Run `synthesize_domain_guides.py` (Layer 4 is optional but strongly recommended) |
| Guide content is hallucinated | Always run synthesis with Qdrant populated — dry-run first to verify chunk retrieval |
| Knowledge pack shadowing variables | Use `kmcp_pack` in `_plan()` if agent's `_do()` already uses `kmcp` for framework access |
