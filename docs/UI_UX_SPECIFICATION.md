# HiMeet Platform UI/UX Specification

> Agent-Centric Design for AI-Powered Marketing Execution

## Design Philosophy

### Core Principles
1. **Agent-First Navigation**: Agents are the primary interaction units, not features
2. **Progressive Disclosure**: Show summary â†’ details on demand
3. **Continuous Value**: Dashboard emphasizes ongoing insights, not one-time reports
4. **Tier-Aware**: UI adapts based on subscription level
5. **Executive-Ready**: Outputs designed for sharing with leadership

### Visual Language
- **Style**: Glassmorphism with depth (continuing current aesthetic)
- **Palette**: Dark mode primary, purple/blue accent gradient
- **Typography**: Clean, professional, scannable
- **Motion**: Subtle, purposeful animations for agent activity

## Responsive Behavior

### Desktop (1440px+)
- Full 3-column layout on Command Center
- Agent Network prominently displayed
- Side-by-side panels in Agent Workspaces

### Tablet (768px - 1439px)
- 2-column layout
- Agent Network collapses to smaller view
- Panels stack vertically

### Mobile (< 768px)
- Single column
- Agent Network becomes horizontal scroll of cards
- Bottom navigation for main sections
- Full-screen modals for detail views

## Accessibility Requirements

1. **Keyboard Navigation**: All interactive elements accessible via Tab
2. **Screen Readers**: ARIA labels on all agent cards and status indicators
3. **Color Contrast**: All text meets WCAG AA standards
4. **Focus Indicators**: Visible focus states on all interactive elements
5. **Motion**: Respect `prefers-reduced-motion` for animations
